# Next.js App Router 가이드

Next.js 15에서는 App Router가 기본으로 제공됩니다. 이 가이드에서는 핵심 개념들을 살펴보겠습니다.

## 프로젝트 구조

App Router는 파일 시스템 기반 라우팅을 사용합니다:

```text
src/
├── app/
│   ├── layout.tsx       # 루트 레이아웃
│   ├── page.tsx         # 홈 페이지
│   └── blog/
│       └── [slug]/
│           └── page.tsx # 동적 라우트
└── components/
    └── Header.tsx
```

## Server Components

React Server Components(RSC)는 기본적으로 서버에서 렌더링됩니다:

```tsx title="app/blog/[slug]/page.tsx"
interface Props {
  params: Promise<{ slug: string }>;
}

export default async function BlogPost({ params }: Props) {
  const { slug } = await params;
  const post = await getPost(slug);

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}

export async function generateStaticParams() {
  const posts = await getAllPosts();
  return posts.map((post) => ({ slug: post.slug }));
}
```

## Data Fetching

Next.js 15에서는 `async` 컴포넌트를 사용하여 데이터를 가져옵니다:

```typescript title="lib/data.ts"
export async function getPost(slug: string) {
  // 파일 시스템에서 읽기
  const content = await fs.readFile(
    path.join(process.cwd(), "content", `${slug}.mdx`),
    "utf-8"
  );

  return parseMarkdown(content);
}
```

## Metadata API

SEO를 위한 메타데이터를 쉽게 설정할 수 있습니다:

```tsx title="app/layout.tsx"
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    default: "My Blog",
    template: "%s | My Blog",
  },
  description: "A minimal markdown blog",
};
```

## 정리

| 기능 | Pages Router | App Router |
|------|------------|------------|
| 렌더링 | Client-first | Server-first |
| 데이터 패칭 | `getStaticProps` | `async` 컴포넌트 |
| 레이아웃 | `_app.tsx` | `layout.tsx` |
| 라우팅 | `pages/` 폴더 | `app/` 폴더 |
| 스트리밍 | ❌ | ✅ |

App Router를 사용하면 더 효율적인 웹 애플리케이션을 만들 수 있습니다.
